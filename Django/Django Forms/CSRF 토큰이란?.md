# 1. CSRF 토큰

Http 통신에서 post 요청은 백엔드 서버에 정보를 변경하는 요청인 것이다. 예를들면 회원가입, 비밀번호 변경, 통장잔고 변경 등이 있다.

해커가 돈을 탈취하기 위해서 post 요청을 위조하는 짓이 있다. post 요청은 html form을 통해서 이뤄지는데 특정인이 해커가 위조한 url을 클릭하는 순간 form이 은행 백엔드에 전송되면서 내가 요청하지도 않았는데 해커에게 돈을 보내라는 요청을 보낼 수 있게된다. 은행은 이를 방지하기 위해서 csrf 토큰을 사용한다.

csrf란? - CSRF(Cross-Site Request Forgery, 사이트간 요청 위조)는 인터넷 사용자가 자신의 의지와는 무관하게 공격자가 의도한 행동을 하도록 만드는 공격 방법입니다. 예를 들어, 당신이 로그인한 상태에서 공격자가 만든 악의적인 링크를 클릭하게 되면, 당신은 모르는 사이에 공격자가 원하는 행동(예: 비밀번호 변경, 메시지 전송 등)을 수행하게 됩니다.

csrf 토큰은 form에 포함되어 있다. 따라서 csrf 토큰이 form에 포함되지 않으면 Django 백엔드는 이를 위조요청이라고 생각하고 post 요청을 거부하게 된다. 이 기능이 django에서 지원하는 csrf 토큰 지원 기능이다.

csrf 토큰은 세션이 유지된 상태에서 보장되는데 세션이 만료되거나 변경 됐을 때 마찬가지로 백엔드 서버는 요청을 거부한다. 만약에 세션이 유지된 상태에서 url을 클릭하여 위조된 form을 전송했다고 하더라도 csrf 토큰에 form에 포함되어 있지 않으면 이 또한 요청이 거부된다.

장고는 csrf 토큰을 지원함으로써 form 위조공격을 방어해준다.
