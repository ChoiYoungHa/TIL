# 5.django ORM 조건 검색

django ORM을 이용해서 db의 데이터를 조회할 때 filter(), get()을 자주 사용하고, 이 메서드들과 함께 사용되는 Q객체는 and, or, not 등 연산자의 조건을 같이 사용할 수 있게 해줌.

**`filter()`와 `get()` 메서드**

- **`filter()` 메서드**: 데이터베이스에서 주어진 조건을 만족하는 모든 레코드를 찾기 위해 사용됩니다. `filter()`는 조건에 일치하는 0개 이상의 인스턴스를 포함하는 쿼리셋(QuerySet)을 반환합니다. 예를 들어, 특정 나이 이상의 모든 환자를 찾으려면 다음과 같이 사용할 수 있습니다:

```python
Patient.objects.filter(age__gte=18)

```

- **`get()` 메서드**: 데이터베이스에서 주어진 조건을 만족하는 단 하나의 레코드를 찾기 위해 사용됩니다. 조건에 일치하는 **레코드가 정확히 하나만 있을 경우 그 인스턴스를 반환**하며, 없거나 여러 개인 경우 예외를 발생시킵니다. 예를 들어, 특정 ID를 가진 환자를 찾으려면 다음과 같이 사용할 수 있습니다:

```python
patient = Patient.objects.get(id=1)

```

**`Q` 객체와 그 기능**

- `Q` 객체는 복잡한 쿼리를 표현하기 위해 사용되며, `filter()`, `exclude()`, `get()` 메서드와 함께 사용됩니다. `Q` 객체는 `|` (OR), `&` (AND), `~` (NOT) 연산자를 사용하여 조건을 결합할 수 있게 해줍니다.
- **기본 사용법**:
    - `|` (OR) 연산자: 두 조건 중 하나라도 만족하는 레코드를 찾습니다.
    - `&` (AND) 연산자: 두 조건을 모두 만족하는 레코드를 찾습니다.
    - `~` (NOT) 연산자: 주어진 조건을 만족하지 않는 레코드를 찾습니다.
- **예시**:

```python
from django.db.models import Q

# 나이가 18세 이상이거나 이름이 'John Doe'인 모든 환자를 찾음
patients = Patient.objects.filter(Q(age__gte=18) | Q(name="John Doe"))

# 나이가 18세 미만이고 이름이 'John Doe'가 아닌 모든 환자를 찾음
patients = Patient.objects.filter(~Q(age__gte=18) & ~Q(name="John Doe"))

```

**추가 기능**

- **`Q` 객체의 중첩**: `Q` 객체를 중첩하여 더 복잡한 쿼리 조건을 만들 수 있습니다. 예를 들어, 여러 조건의 조합을 사용하여 특정한 패턴의 쿼리를 생성할 수 있습니다.
- **다이내믹 쿼리 생성**: `Q` 객체를 사용하면 쿼리 조건을 프로그래매틱하게 생성하고 조합할 수 있습니다. 이는 런타임에 쿼리 조건을 결정해야 할 때 유용합니다.

`Q` 객체는 Django의 쿼리 구성에서 유연성과 표현력을 크게 확장해주는 강력한 도구입니다. 이를 통해 개발자는 데이터베이스 상의 복잡한 조회 조건들을 Pythonic하게 표현할 수 있으며, 이는 Django 애플리케이션의 데이터 처리 능력을 대폭 강화시킵니다. `Q` 객체의 추가적인 기능과 사용 사례에 대해 계속해서 탐구해 보겠습니다.

**`Q` 객체의 활용성 확장**

- **조건의 동적 조합**: `Q` 객체를 사용하면 쿼리 조건을 동적으로 조합하여 생성할 수 있습니다. 예를 들어, 사용자 입력에 따라 검색 조건을 바꿔야 할 때, `Q` 객체를 활용하여 필요한 조건을 프로그래매틱하게 추가하고 조합할 수 있습니다.
- **고급 검색 기능 구현**: 사용자로부터 여러 필터링 조건을 받아 복잡한 쿼리를 구성해야 하는 고급 검색 기능에서 `Q` 객체는 특히 유용합니다. 예를 들어, 사용자가 여러 조건을 조합하여 특정 레코드를 찾기 원하는 경우, 이러한 조건들을 `Q` 객체를 통해 효율적으로 처리할 수 있습니다.
- **조건의 부정 사용**: `~` 연산자를 사용하여 특정 조건을 부정하는 쿼리를 쉽게 생성할 수 있습니다. 이는 특정 조건을 제외한 결과를 얻고자 할 때 유용합니다.

**예제: 동적 쿼리 생성**

```python
from django.db.models import Q

# 사용자로부터 받은 검색 조건
search_age = 25
search_name = "John Doe"

# 초기 Q 객체 생성
query = Q()

# 조건 추가
if search_age:
    query |= Q(age__gte=search_age)
if search_name:
    query |= Q(name__icontains=search_name)

# 최종 쿼리 실행
patients = Patient.objects.filter(query)

```

위 예제는 사용자로부터 받은 나이와 이름 조건을 바탕으로 동적으로 `Q` 객체를 구성하고, 이를 사용하여 최종 쿼리를 실행하는 방법을 보여줍니다. 이처럼 `Q` 객체를 활용하면, 런타임에 조건을 결정하고 쿼리를 구성하는 유연성을 얻을 수 있습니다.

`Q` 객체는 Django ORM에서 제공하는 강력한 기능 중 하나로, 복잡한 데이터베이스 쿼리를 간결하고 효율적으로 표현할 수 있게 해줍니다. 이를 통해 데이터에 대한 깊은 통찰과 다양한 조건에 맞는 정교한 데이터 접근 방식을 구현할 수 있습니다.
