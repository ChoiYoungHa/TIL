# 2. django 데이터베이스 연동하기_테이블 생성

django 프로젝트 생성

```bash
$ django-admin startproject <project name>
```

생성된 프로젝트로 이동 후

```bash
$ djnago-admin startapp <app name>
```

python에 내장된 데이터베이스인 sqlite 생성

```bash
$ python manage.py migrate
```

<project_name>/db.sqlite 가 생성 됨

<project_name>/<app_name>/apps.py 이동 후 예제 테이블 생성

```python
from django.db import models

class Patient(models.Model):
  first_name = models.CharField(max_length=30)
  last_name = models.CharField(max_length=30)
  age = models.IntegerField()
```

변경된 내용 적용

```python
$ python manage.py makemigrations <app_name>
```

예제 테이블이 sql문으로 변환되기 위한 청사진 파일로 변환 됨.

<project_name>/<app_name>/migrations/0001_initial.py

```python
# Generated by Django 4.2.11 on 2024-03-11 09:06

from django.db import migrations, models

class Migration(migrations.Migration):

    initial = True

    dependencies = [
    ]

    operations = [
        migrations.CreateModel(
            name='Patient',
            fields=[
                ('id', models.BigAutoField(auto_created=True, primary_key=True, serialize=False, verbose_name='ID')),
                ('first_name', models.CharField(max_length=30)),
                ('last_name', models.CharField(max_length=30)),
                ('age', models.IntegerField()),
            ],
        ),
    ]

```

해당 파일을 sql문으로 변환해서 보고 싶다면

```python
$ python manage.py sqlmigrate <app_name> 0001
```

**결과**

```python
$ python manage.py sqlmigrate office 0001
BEGIN;
--
-- Create model Patient
--
CREATE TABLE "office_patient" ("id" integer NOT NULL PRIMARY KEY AUTOINCREMENT, 
"first_name" varchar(30) NOT NULL, "last_name" varchar(30) NOT NULL, 
"age" integer NOT NULL);
COMMIT;
```

해당 sql문을 slqite에 적용

```bash
$ python manage.py migrate <app_name>
```

테이블 생성완료

```python
$ python manage.py migrate office

Operations to perform:
  Apply all migrations: office
Running migrations:
  Applying office.0001_initial... OK
```

### 사용된 명령어 정리

1. **`python manage.py migrate`**
    - **목적**: 이 명령어는 마이그레이션 파일들을 실제 데이터베이스에 적용합니다. Django 프로젝트의 모델 변경사항에 대해 생성된 마이그레이션 파일들을 데이터베이스 스키마에 반영하여, 모델의 현재 상태를 데이터베이스 스키마와 동기화시킵니다.
2. **`python manage.py makemigrations`**
    - **목적**: 프로젝트의 모델에 변경사항이 있을 때, 이 변경사항을 추적하여 새로운 마이그레이션 파일을 생성합니다. 이 파일은 모델의 변경사항을 데이터베이스 스키마 변경 명령어로 변환한 것이며, **`migrate`** 명령어를 사용하여 데이터베이스에 적용할 수 있습니다.
3. **`python manage.py sqlmigrate <app_label> <migration_name>`**
    - **목적**: 특정 마이그레이션 파일이 데이터베이스에 적용될 때 실행되는 SQL 문을 출력합니다. 이 명령어는 마이그레이션이 실제로 어떤 SQL 명령어를 실행하는지 이해하고자 할 때 유용하며, 데이터베이스 관리와 디버깅 과정에서 도움을 줍니다.
4. **프로젝트 생성**: **`django-admin startproject <projectname>`**
    - **목적**: 새로운 Django 프로젝트를 생성합니다. 이 명령어는 프로젝트 디렉토리와 기본 설정 파일들을 생성합니다.
5. **앱 생성**: **`django-admin startapp <appname>`**
    - **목적**: 프로젝트 내에 새로운 앱을 생성합니다. 각 앱은 프로젝트의 특정 기능을 담당합니다. 이 명령어는 앱 디렉토리와 함께 기본 파일들을 생성합니다.
