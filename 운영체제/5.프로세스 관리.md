# 5. 프로세스 관리

[ 프로세스의 상태 ]

- 프로세스는 각 상태가 있다.
- 메모리에 올라간 상태 new
- 초기화를 끝내고 실행준비 완료인 상태 ready
- cpu가 프로세스를 실행중인 상태 running
- 프로세스가 실행중에 io를 만났을 때 waiting
- IO를 사용하고 있는상태에서는 해당정보를 하드에 보내고 프로세스는 다른일을 처리하기 때문에 해당 프로세스는 다시 cpu서비스를 기다리는데 그 상태를 waiting 상태라고 함.
- 그리고 터미네이티드 terminated 프로세스가 종료된 상태

[ 프로세스의 상태중 타임쉐어링만의 프로세스 상태 ]

- 타임 쉐어링 시스템은 cpu가 특정 프로세스의 어느시간이 경과하면 바로 넘겨버리는데 이를  타임 익스피디어 라고하는데 이상태가 러닝이었다가 바로 레디로 바뀌는 상태다.

[ PCB란? ]

- PCB는 프로그램 컨트롤 블록이라고해서 프로세스의 모든 정보가 저장되어있는 블록을 말함.
- 이 블록에는 프로세스 상태, 프로그램카운터, 레지스터주소, 메모리의 LIMIT BASE값, PID등이 저장되어있음
- 프로세스를 스케줄링하면서 여러개의 프로세스를 CPU가 와따리가따리 사용하는데 원래 프로세스로 되돌릴 때 필요한 정보들이 PCB에 저장되어있음.

[ PID란? ]

- 그냥 프로세스 번호

[ JOB 스케줄러 ]

- LONG TERM 스케줄러라고도하며, 특정 메모리가 비워졌을 때 메모리에 올릴 수 있기때문에 시간이 오래걸린다.
- HDD에서 메모리에 올라가기까지 대기하는 공간을 JOB-Q라고하는데 그곳을 관리하는 녀석이 JOB 스케줄러다.

[ CPU 스케줄러 ]

- 이 친구는 메모리 올라가있는 프로세스들을 관리하는 친구다. 메모리에 어떤 프로세스부터 실행할지 관리하는 친구다.
- CPU는 1초에 100번 스위칭하기때문에 모든 프로세스들을 관리한다. 프로세스에게 다시 신호를 주는 텀이 짧다. 그래서  SHOURT TERM 스케줄러라고도 한다.
- 메모리에 있는 프로세스가 CPU의 서비스를 받기까지의 대기열을 READY-Q 라고한다. 이 READY-Q의 우선순위를 관리하는 친구가 바로 CPU 스케줄러다.

[ 디바이스 스케줄러 ]

- 프린터, 마우스, 하드사용 등 여러 입출력 대기열을 DIVICE-Q 라고하는데 대기열의 우선순위를 관리하는 스케줄러다.

[ DEGREE OF MULTIPROGGRAMING ]

- DEGREE는 수준을 의미함. 멀티 프로그래밍은 메모리에 프로세스가 여러개 돌아가는 것을 의미하는데 이 메모리에 얼마나 많이 올라가느냐의 척도를 디그리 오브 멀티프로그래밍이라고 함.
- 멀티 프로그래밍의 정도. 즉,수행중인 프로세스의 수

[ 각 프로세스의 특징 ]

- 입출력을 많이하는 프로세스 IO 바운드 프로세스
- 계산을 많이하는 CPU 바운드 프로세스
- 이 프로세스들이 하드에서 메모리로 올라갈 때 조금 골고루 올라가야 성능이 나오는데 이때 작업을 JOB스케줄러가 한다.

[ 미디엄텀 스케줄러 ]

- 대표적으로 TIME SHARING SYSTEM 즉 대화형시스템에서 많이 사용되는 스케줄러임.
- 실시간으로 OS가 메모리를 지켜보다가 한 사용자가 잠깐 자리를 비운거 같다면 그 사람 때문에 할당했던 메모리를 잠깐 하드이미지 파일로 옮기고 그 메모리를 사용함.
- 이렇게 메모리에서 잠깐 사용하지 않는 녀석을 하드정보로 바꿔서 쫒아내는 작업을 SWAP OUT 이라함.
- 그 메모리의 정보를 옮겨서 저장하는 하드의 장소를 SWAP DIVICE라고함.
- 그리고 사용자가 다시 돌아왔을 때 하드에 저장된 메모리의 기억정보를 다시 가지고와서 활성화 시키는 작업을 SWAP IN 이라고함.
- 이런식으로 너무 길지도 짧지도 않게 일어나는 메모리 스왑현상을 관리한다고해서 미디엄텀 스케줄러임.
- 이러한 SWAP OUT SWAP IN이 반복되는 운영방식을 스왑핑(SWAPPING) 이라고 함.
