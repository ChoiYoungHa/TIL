# 2. 고등 운영체제

**[ 컴퓨터 구조 ]**

- 컴퓨터는 폰노이만 구조로 cpu 메모리가 버스로 연결되어있는 구조다.

**[ 컴퓨터 구조에 맞는 OS ]**

- 우리가 여러개의 cpu를 사용해서 컴퓨터의 성능을 높힌다.
- 여러개의 cpu를 사용한다면 그에 맞는 os가 필요하다.
- 그 os의 이름은 다중 프로세스 시스템이다. 이것은 고등 운영체제에 해당한다.
- multiprocessor system, 병렬 시스템, 강결합 시스템이라고도한다.
- 강결합인 이유는 메모리와 cpu가 버스로 강하게 직접연결 되어 있기 때문이다.
- 이렇게함으로써 성능개선, 신뢰성, 비용절감 효과를 누리게 된다.

**[ 컴퓨터 구조에 맞는 OS 2]**

- 우리는 여러개의 cpu를 사용하기도 하지만 여러 컴퓨터를 lan으로 묶어서 분산하여 사용하기도 한다.
- 하지만 이아이들은 하나의 일을 처리하기 위해 연결되어있다. 예를들어 일기예보라는 일을 처리하기 위해서는 각각 한 컴퓨터당 하나씩 지역을 맡고 그 지역마다의 날씨는 다른 지역에도 영향을 주기 때문에 서로 내용을 message로 공유해야한다.
- 마치 팀프로젝트 같은 일을 하는 것인데 이러한 일을 하기 위해서는 일단 os구조가 달라야한다.
- 이 운영체제를 분산시스템 Distributed System 이라고 한다. 또는 소결합 시스템이라고도 한다.
- 소결합인 이유는 메모리와 cpu가 직접연결된 것이 아닌 lan을 통해 서로 연결되어 있기 때문에 약한 결합 이라고 하기 때문에 소결합이라고 표현한다.
- 멀티 컴퓨터 시스템이라고도 한다.

**[ 컴퓨터 구조에 맞는 OS 3]**

- 이번건 컴퓨터 구조에 따라 달라지는 것이 아니고 상황에 따라 달라지는 운영체제라서 조금은 특별하게 느껴질 수 있다.
- Real Time System  실시간 시스템은 어떤 계산이 정해진 시간내에 반드시 끝나야할 때 실시간 시스템이라고 한다. 주로 네비게이션, 공장자동화, 군사목적으로 사용된다.
- 정해진 시간내에 정확한 계산이 이루어져야하기 때문이다. 네이게이션은 교차로를 통과할때까지 경로를 계산 공장은 특정부분에 용접이 언제까지 끝나고 다음작업 연결 군사목적은 우리 진영에 미사일이 떨어지기 전까지 계산끝내고 격추 이러한 목적이다.
- 이럴려면 OS가 어떻게 달라야할까? 프로세스 p1 p2 p3 가 있다고할 때 p2가 데드라인을 못지킬거 같다이러면 운영체제가 p2를 감시 관리하면서 p1 p3를 버리는 한이 있더라도 p2가 원래 시간동안 동작할 수 있도록 도와주는 형태이다.

**[ 인터럽트 ]**

- H/W 인터럽트는 우리가 마우스를 클릭했을 때 전기신호가 cpu의 인터럽트 선을 건들이면서 cpu는 인터럽트를하고 os에 내장되어있는 파일로 점프해서 마우스 인터럽트 루틴을 실행시킨다. 그 코드는 수직선상에 내 마우스의 위치 x y를 실시간으로 계산하여 표현해준다. 또한 키보드를 입력했을 때도 마찬가지로 os로 점프해서 해당하는 코드파일을 실행시켜 지금처럼 입력을 도와주는 기능 구현한다.
- S/W 인터럽트는 우리가 만약에 유저프로그램을 실행중에 갑자기 이메일이 확인하고 싶어서 인터넷을 킨다하면 그때 소프트웨어 인터럽트가 실행되면서 os안에있는 코드파일을 실행시키고 그 코드가 하드에 인터넷실행파일을 찾고 그것을 실행시키고 그런담에 모든게 끝나면 원래 프로그램으로 돌아간다.
- 이러한 루틴들이 프로그램 실행 / 인터럽트 후 되돌아오기 / 프로그램 실행 / 인터럽트 후 되돌아오기를 반복하면서 실행한다. 그래서 현대의 운영체제는 인터럽트 기반 시스템이라고 볼 수 있다.]
- 내부 인터럽트는 5 / 0 을 나눌려고할 때 5에서 0을 나눌 수가 없기때문에 저장할 공간이 없어 밑으로 코드를 진행할 수 없어서 Divide by zero 즉 0으로 나눴을 때 그 명령을 인터럽트로 종료시키라는 의미
