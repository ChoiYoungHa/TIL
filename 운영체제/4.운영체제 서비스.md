# 4. 운영체제 서비스

[ CPU 보호 ] - 메모리보호, IO보호, CPU보호 그 중 하나임.

- CPU가 하나의 프로세스에게 독점 당하지 않으려면 어떻게 해야할까?
- 만약 하나 프로그램이 while(n = 1) 반복문이 실행중이라고 해보자. 그러면 이 프로세스가 CPU를 독점함
- 이 문제는 타이머를 사용해서 100m/s 마다 인터럽트를 실행시켜서 하나의 프로세스에게 먹혔는지 아니면 잘 돌아가고있는지 감시한다. 만약 프로세스하나가 독점 중이라면 ISR 인터럽트 서비스 루틴을 OS에게 실행을 부탁하고 OS는 CPU에게 해당 루틴을 실행을 하게 해서 해당 독점 프로세스를 강제 종료시킨다.

[ 운영체제가 정부고 각 역할을 담당하는 부서들이 있다 ]

- 프로세스 매니지 먼트 (PROCESS MANAGE MENT)
    - 주요 기능으로 프로세스 생성, 소멸 , 일시중지 , 활동재개
    - 프로세스간의 통신, 동기화, 교착상태 처리(Dead Lock)
- 메인 메모리 매니지 먼트
    - 프로세스에게 메모리 공간할당, 메모리 어느부분이 어느 프로세스에게 할당 되었는가 추적감지, 프로세스 종료 시, 메모리 회수
    - 메모리의 효과적 사용, 가상 메모리 물리적 실제 메모리보다 큰 용량을 갖도록
- 파일 매니지 먼트
    - 원래 파일은 트랙, 섹터로 나뉘어져있음, 이것들을 file 단위로 볼 수 있는게 운영체제 덕분임
    - 파일생성 삭제, 디렉터리 생성 삭제, 기본 동작지원(open, close, read, write, create, delete)
    - Track / Sector 간의 File 맵핑 그리고 빽업
- 보조기억장치 관리
    - 디스크에 트랙, 섹터들이 모인애들이 블락인데 이 블락에 데이터를 저장함. 빈공간 관리, 사용되는 공간관리, 하드, 플래시 메모리등
    - 저장공간 할당, 디스크 스케줄링 역할
- I/O 장치 관리
    - 장치 드라이브(새로운 하드웨어의 드라이브)관리,
    - 입출력 장치의 성능 향상 (Buffering, cashing, spooling)
    - 스플링은 프린터와 cpu 속도가 차이가 많이 나니 정보를 하드에게 준담에 천천히 하드에있는 내용을 프린터에게 전달할 수 있도록 함.

- System Calls 운영체제 서비스를 받기 위한 호출
    - P1, P2 프로세스가 돌다가 필요한 작업에 대해 OS에게 작업을 요청하는 것이 System calls이다.
    - 시스템콜은 특정 레지스터에 특정값을 주고 소프트웨어 인터럽트를 걸어줌으로서 서비스를 받을 수 있다.
    - 원래 과거의 MS-DOS에서는 이러한 동작들을 전부 어셈블리어로 작성해서 컴파일하고 컴파일 한것 끼리 C언어 파일이랑 링킹하고 오브젝트 파일 만들어지면 그거 실행하고 그 실행 동작이 시스템 콜이다.
    - 지금의 운영체제는 정말 간편해진 것
